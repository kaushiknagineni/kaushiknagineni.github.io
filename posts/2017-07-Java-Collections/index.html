Iterator Interface
Generics added type safety feature that collections had been missing.
Autoboxing and unboxing facilitates the storing of primitive types in collections.
The colection interfaces defines several core interfaces: Collection, List, Set, Queue, SortedSet, NavigableSet, and Deque

Collection is a generic interface that has this declaration: interface Collection where E specifies the type of Objects the collection will hold. Collection extends the Iterable interface

Methods declared by Collection

  • boolean add(E obj) - adds obj to the invoking collection. Returns true if obj was added to the collection. Returns false if obj is already a member of the collection.
  • boolean addAll(Collection< extends E> c) - Adds all the elements of c from the invoking collection. Returns true if the invoking collection is changed, else returns false
  • void clear() - removes all elements from the invoking collection
  • boolean contains(Object obj) - Returns true if obj is in the invoking collection. Otherwise, returns false.
  • boolean containsAll(Collection<> c - Returns true if the invoking collection contains all the elements of collection c
  • boolean equals(Object obj) - returns true if the invoking collection and obj are same.
  • int hashCode() - returns the hash code for the invoking collection.
  • boolean isEmpty() - returns true if the invoking collection is empty. Otherwise, false
  • Iterator iterator() - Returns an iterator for the invoking collection
  • boolean remove()
  • boolean removeAll()
  • boolean retainAll(Collection<> c) - retains all the elements in the invoking collection that are present in c. Returns true if the collection changed. Otherwise, returns false.
  • int size - returns the number of elements in the invoking collection.
  • Object[] toArray() - Returns an array that contains the elements in the invoking collection. The array elements are copies of collection elements.
  • T[] toArray(T array[])

Obtaining an array from an ArrayList

Reasons to convert a collection into an Array 1) To obtain faster processing times for certain operations 2) To pass Array to a method that is not overloaded to accept a collection 3) To interate collection based code with legacy code that do not understand collections. We use toArray() to convert a collection to an Array. The most common way is: T[] toArray(T array[]) - this returns an array of elements that have same type as T The other way is: Object[] toArray()

The HashSet class

HashSet implements the Set interface. Class HashSet A hash table stores information using a mechanism called hashing. In hashing, the informational content of a key is used to determine a unique value, called its hashcode. The hashcode is then used to index at which the data associated with the key is stored. The advantage of hashing is that it allows the execution time of add(), contains(), remove(), and size() to remain constant even for large sets. The following constructors are: HashSet() HashSet(Collection< extends E> c) HashSet(int capacity) HashSet(int capacity, float fillRatio) HashSet doesn't define any additional methods beyond those provided by its superclasses and subclasses HashSet doesn't guarantee the order of its elements, because the process of hashing doesn't ususally lend itself to the creation of sorted sets. If we need sorted storage, TreeSet is a better choice.

TreeSet Class

TreeSet implements the NavigableSet interface. It creates a collection that uses the tree for storage. Objects are stored in sorted ascending order. Class TreeSet TreeSet() - constructs an empty tree set that will be sorted in ascending order. TreeSet(Collection< extends E> c) - builds a tree set that contains elements of c. TreeSet(Comparator< super E> comp) - Constructs an empty tree set that will be sorted according to the constructor specified by comp TreeSet(SortedSet ss) - builds a tree set that contains the elements of ss.

Working with Maps

Maps don't implement the Iterable interface. This means we can't iterate through the map using the conventional for loop. The following interfaces supports Maps. Map - Maps unique keys to values Map.Entry - Describes an element(a key/value) pair in the Map. SortedMap - Extends Map so that keys are maintained in ascending order.

The Map Interface

Maps revolve around two basic operations get() and put(). To obtain a collection view of the map we use entrySet method. entrySet returns a Set that contains elements in the map. To obtain a collection view of the keys, we use keySet(). To obtain a collection view of the values, we use values(). Collection views are the means by which maps are integrated into the larger Collections Framework.
